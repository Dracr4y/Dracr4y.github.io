在github上部署hexo博客

  1. 需要安装的工具：Git，Node.js

  2. 右键打开git

  3. 全局安装hexo

  4. 在想要安装的文件夹下打开git，执行hexo init

  5. 安装依赖

  6. 启动服务

  7. 访问localhost:4000,显示以下内容即成功

  8. 注册github账号

  9. 创建仓库【名字后边应跟上.github.io】

  10. 配置ssh key，连续三次回车，会生成密钥在本地

  11. 打开复制的内容，cd ~/.ssh，然后cat id_rsa.pub

  12. 再打开你的github设置，把刚才复制的内容放进去，标题随便填

  13. 测试是否成功

  14. 配置用户名和邮箱

  15. 配置hexo的_config.yml文件【记得加空格！！！大坑】

  16. 安装一个插件（不然会报错）

  17. 现在可以上传你的网页到github了，执行hexo d

  18. 访问你的github page网址

  19. Done

    20.（2020年4月27日补充）我本来的md文章因为是石墨文档直接导出的，图片链接是石墨的服务器，导致加载不出，网上搜了一大堆“如何更改文章”也搜不到，于是自己摸索。

首先找到未渲染的源文件

我把全部的图片都换成了本地相对地址。（石墨导出为word文档然后复制粘贴过来）

后来上传后发现相对地址也不能识别，因为渲染完了没把图片上传上去。

于是想着在source目录下新建个image文件夹，把图片都放进去，然后把md的图片换成/image/…这样的格式，本地是显示不出的。

然后hexo g重新生成，hexo d 上传，OK.

这就是我不想用hexo的原因，处理图片太麻烦了。

【题外话】

时隔一年这篇最初的文章终于可以显示图片，本来有了新博客我都懒得理了，结果被同学挑起了兴趣，就一股脑解决吧。
